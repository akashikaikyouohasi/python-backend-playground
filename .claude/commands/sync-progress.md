# 進捗同期

PROGRESS.mdと実際のファイル構造を照合し、進捗状況を自動更新してください。

## 実行手順

1. **現在のファイル構造を確認**
   - プロジェクトルートのファイル一覧
   - `src/` ディレクトリ構造
   - `tests/` ディレクトリ構造
   - `docs/adr/` のADR一覧
   - `terraform/` ディレクトリ（存在する場合）
   - `.github/workflows/` ディレクトリ（存在する場合）

2. **PROGRESS.mdを読み込み**
   - 各タスクの現在の状態を確認
   - 成果物カラムを確認

3. **照合と更新**
   - 成果物ファイルが存在するタスク → ✅に更新
   - 成果物ファイルが部分的に存在 → 🚧に更新
   - ファイルパスを成果物カラムに記入
   - 完了日を追加（gitログから取得可能なら）

4. **pyproject.tomlの確認**
   - 導入済みの依存関係を確認
   - 対応するタスクを更新

5. **設定ファイルの確認**
   - `.pre-commit-config.yaml` → pre-commitタスク
   - `pyrightconfig.json` → Pyrightタスク
   - `Makefile` → Makefileタスク
   - `docker-compose.yml` → Docker Composeタスク
   - `alembic.ini` → Alembicタスク

## 出力

1. 変更内容のサマリーを表示
2. PROGRESS.mdを実際に更新
3. 不整合があれば報告（ファイルはあるがタスクがない、など）

## 注意事項

- 既に✅のタスクは変更しない（ユーザーが手動で完了にした可能性）
- 不確実な場合は🚧（進行中）にする
- 大きな変更前に確認を求める
