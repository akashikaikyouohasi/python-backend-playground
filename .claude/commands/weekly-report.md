# 週次レポート生成

今週の学習進捗をサマリーとして生成します。

## 情報収集

1. **Gitログを確認**
   - 過去7日間のコミット履歴
   - 変更されたファイル

2. **PROGRESS.mdの変更を確認**
   - 今週完了したタスク（⬜→✅）
   - 新しく開始したタスク（⬜→🚧）
   - 更新された成果物

3. **ADRを確認**
   - 今週作成されたADR
   - 重要な技術判断

4. **コード量を確認**
   - 追加された行数
   - 削除された行数
   - 新規ファイル数

## レポート形式

```markdown
# 週次レポート: YYYY-MM-DD 〜 YYYY-MM-DD

## サマリー
- 完了タスク: X件
- 進行中タスク: Y件
- 作成ADR: Z件

## 今週の成果

### 完了したタスク
1. [タスク名] - 成果物: `ファイルパス`
2. ...

### 作成した成果物
- `path/to/file1`
- `path/to/file2`

### 技術判断（ADR）
- ADR-XXX: タイトル

## コード統計
- コミット数: N
- 追加行数: +XXX
- 削除行数: -XXX

## 学んだこと
_(PROGRESS.mdの「学んだこと」セクションから抽出)_

## 来週の予定
_(PROGRESS.mdの次の未完了タスクから推測)_

## 課題・ブロッカー
_(進行中タスクが長期間停滞している場合に記載)_
```

## 出力

1. レポートを表示
2. オプション: `docs/reports/weekly-YYYY-MM-DD.md` として保存するか確認
3. PROGRESS.mdの「学んだこと」セクションに追記を促す
