# 次のタスク提案

PROGRESS.mdを分析し、次に取り組むべきタスクを提案してください。

## 分析手順

1. **現在の状態を確認**
   - PROGRESS.mdを読み込み
   - 進行中（🚧）のタスクを特定
   - 現在のPhaseを確認

2. **依存関係を考慮**
   - 前提タスクが完了しているか確認
   - 例: Alembic設定前にSQLAlchemy設定が必要

3. **優先度を評価**
   - LEARNING_PLAN.mdの重点分野（DBパフォーマンス、API設計）を考慮
   - ブロッカーになりそうなタスクを優先
   - 小さく始められるタスクを推奨

## 出力形式

```
## 推奨: 次のタスク

### 1. [タスク名]
- **理由**: なぜこのタスクを次にやるべきか
- **前提条件**: 必要な準備（あれば）
- **成果物**: 期待されるアウトプット
- **推定時間**: 目安時間
- **開始コマンド**: 最初に実行するコマンドやアクション

### 代替案

もし上記が難しい場合:
- タスクA: 理由
- タスクB: 理由
```

## 注意事項

- 進行中のタスクがあれば、まずそれを完了させることを推奨
- 1回に1つのタスクに集中することを推奨
- タスクが大きすぎる場合は分割を提案
